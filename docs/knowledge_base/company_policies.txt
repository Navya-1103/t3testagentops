1. DATA HANDLING POLICY
=============================================================================

PURPOSE: Protect customer and personal data through proper storage and access controls.

DATA CLASSIFICATION:
- Public: Marketing materials, published reports (no restrictions)
- Internal: Business plans, internal communications (standard access controls)
- Confidential: Customer contacts, employee records (encryption + logging required)
- Restricted: Customer PII, payment data, health records (encryption + approval required)

APPROVED STORAGE:
✓ Company OneDrive/SharePoint
✓ Approved encrypted databases
✓ Encrypted local drives (temporary only)

PROHIBITED STORAGE:
✗ Personal email accounts
✗ Personal cloud storage (Dropbox, Google Drive personal)
✗ Unencrypted USB drives
✗ Shared Google Sheets (for customer data)

KEY RULES:
- Customer data MUST NOT be stored in shared Google Sheets
- Two-factor authentication (2FA) required for all customer data systems
- Downloads of restricted data require approval
- Maximum 30 days retention for local copies
- All access to restricted data is logged

INCIDENT REPORTING:
- Report suspected breaches within 1 hour
- Contact: security@company.com or x5555
- Do not investigate on your own

2. THIRD PARTY ACCESS POLICY
=============================================================================

PURPOSE: Control sharing of data and systems with external vendors and contractors.

APPROVAL REQUIREMENTS:

Low Risk (No customer data):
- Department manager approval
- Timeline: 5 business days

Medium Risk (Internal data only):
- Manager + IT Security + Procurement
- Timeline: 10 business days

High Risk (Customer data access):
- Manager + CISO + DPO + Legal + Executive
- Timeline: 20-30 business days

US VENDORS:
- SOC 2 Type II certification preferred
- Privacy Shield compliance if applicable
- Insurance verification ($1M minimum)
- Background checks for key personnel

EU VENDORS:
- GDPR compliance certification required
- Standard Contractual Clauses (SCCs) required
- Data Processing Agreement (DPA) required
- Additional Legal and DPO approval required

CONTRACTOR ONBOARDING CHECKLIST:
1. Background check completed
2. NDA signed
3. Security training completed
4. Acceptable use policy acknowledged
5. Manager approval obtained
6. IT Security approval obtained
7. Access provisioned with expiration date

ACCESS CONTROLS:
- Unique credentials (no shared accounts)
- Multi-factor authentication required
- Least privilege access
- Time-limited (max 1 year, then renewal)
- VPN required for remote access

3. SECURITY INCIDENT REPORTING POLICY
=============================================================================

PURPOSE: Ensure quick response to security incidents to minimize impact.

INCIDENT SEVERITY:

Critical (P1) - Response within 15 minutes:
- Customer PII exposed or stolen
- Ransomware affecting critical systems
- Active data exfiltration
- Notify: CISO, CEO, Legal, PR

High (P2) - Response within 1 hour:
- Suspected data breach under investigation
- Malware on multiple systems
- Unauthorized access to sensitive systems
- Notify: CISO, IT Security Manager, Legal

Medium (P3) - Response within 4 hours:
- Isolated malware infection
- Failed unauthorized access attempts
- Policy violations
- Notify: IT Security Team, Department Manager

WHAT TO DO IF YOU ACCIDENTALLY EXPOSE CUSTOMER PII:

IMMEDIATE ACTIONS (within 1 hour):
1. Call Security Hotline: x5555 immediately
2. Do not access the exposed data further
3. Preserve evidence (screenshots, logs)
4. Document what data was exposed
5. Note who may have accessed it

DO NOT:
- Try to investigate on your own
- Delete or modify evidence
- Discuss publicly or on social media
- Continue using compromised systems

PHISHING EMAIL RESPONSE:
1. Do not click links or open attachments
2. Report via "Report Phishing" button
3. Or forward to phishing@company.com
4. Delete after reporting

IF YOU CLICKED A PHISHING LINK:
1. Immediately call x5555
2. Disconnect from network
3. Do not log into any accounts
4. Security will reset your credentials

LOST/STOLEN DEVICE:
1. Call Security Hotline x5555 within 1 hour
2. Report to local police if stolen
3. Provide device serial number and last location
4. Security will initiate remote wipe

4. ACCEPTABLE USE POLICY
=============================================================================

PURPOSE: Define appropriate use of company systems and resources.

GENERAL PRINCIPLES:
- Systems provided for business use
- Limited personal use acceptable (breaks/lunch)
- All activity may be monitored
- Users responsible for all activity under their accounts

PASSWORD REQUIREMENTS:
- Minimum 12 characters
- Mix of uppercase, lowercase, numbers, symbols
- Changed every 90 days
- No password reuse (last 12 passwords)
- Never share passwords

MULTI-FACTOR AUTHENTICATION (MFA) REQUIRED FOR:
- VPN access
- Email access (remote)
- Cloud applications
- Systems containing customer data
- Financial systems

ACCEPTABLE USE:
✓ Business communications
✓ Work-related research
✓ Professional development
✓ Reasonable personal use during breaks

PROHIBITED:
✗ Illegal activities
✗ Harassment or discrimination
✗ Inappropriate content
✗ Unauthorized access attempts
✗ Circumventing security controls
✗ Installing unauthorized software
✗ Cryptocurrency mining
✗ Using company resources for personal business

SOFTWARE INSTALLATION:
- Only approved software may be installed
- Request through IT Service Portal
- No pirated or unlicensed software
- IT approval required for all installations

REMOTE ACCESS:
- VPN required for all remote access
- MFA required for VPN
- Only from secure networks
- No public WiFi without VPN
- Keep remote systems updated

CLOUD SERVICES:
Approved:
✓ Microsoft OneDrive (company account)
✓ SharePoint Online
✓ Approved SaaS applications

Prohibited:
✗ Personal cloud storage for work documents
✗ Unauthorized file sharing services
✗ Personal email for work documents

MONITORING:
Company monitors:
- Network traffic
- Email communications
- Internet usage
- File access and transfers
- Login attempts
- Application usage

VIOLATIONS:
- First: Written warning + retraining
- Second: Performance review + access restrictions
- Serious: Immediate termination + legal action
